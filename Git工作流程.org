* 概述
多个人参与同一个项目开发时，git add commit push 一把梭的方式，是不行的，
这里结合前辈的经验和自己的理解，结合场景说明白这个工作流程。

* 工作流程
场景说明：
- 主分支是 main
- 张三和其它若干人合作开发

工作流程
1. 张三将代码 git clone 到本地，在自己的开发分支 dev_zhangsan 上修改代码
   git clone https://xxx.com/abc.git
   git checkout -b dev_zhangsan
2. 代码修改完成后，git add commit 将代码及其修改记录保存到本地 git 中
   git add .
   git commit -m '修改记录'
   git push origin dev_zhangsan
3. 此时，main 分支上有别人修改的内容，将 main 上的内容合并到 dev_zhangsan 上
   git checkout main
   git pull origin main
   git checkout dev_zhangsan
   git rebase main
   git push -f origin dev_zhangsan
4. git Web 页面上提交，合并
   New pull request
   Squash and merge
5. 删除本地分支
   git checkout main
   git branch -D dev_zhangsan
